# Création d’un ticket dans GLPI

###  Objectif
Ce document décrit la procédure de **création d’un ticket** dans GLPI,
du point de vue d’un **utilisateur final**.
La création correcte d’un ticket est essentielle pour permettre
une prise en charge efficace par l’équipe support IT.



## 1.Contexte

La création d’un ticket permet à un utilisateur :
- De signaler un incident ou une demande
- De décrire précisément un problème
- De déclencher le processus de support IT


### Connexion utilisateur
- URL :
```arduino
http://<IP_SERVEUR>/
```
- Connexion avec un compte **profil Utilisateur**

### Description du problème

#### Type :
- Incident
- Demande

#### Catégorie :
Exemples :
- Réseau → Perte de connexion Internet
- Matériel → Poste ne démarre pas
- Logiciel → Application non fonctionnelle

#### Urgence :
- Faible
- Moyenne
- Haute
- Critique

#### Titre
Description courte et claire du problème  
Exemple :
> Perte de connexion Internet sur poste bureautique

#### Description
Description détaillée du problème :
- Symptômes observés
- Depuis quand
- Messages d’erreur éventuels
- Actions déjà tentées


## 2. Ajout de pièces jointes

L’utilisateur peut joindre :
- Capture d’écran
- Photo
- Document

Cela facilite le diagnostic par le support IT.


## 3. Validation et création du ticket

- Vérifier les informations saisies
- Cliquer sur **Envoyer le message**

Résultat :
- Le ticket est créé
- Un numéro unique est attribué
- Le statut initial est **Nouveau**


##  4. Suivi du ticket par l’utilisateur

L’utilisateur peut :
- Consulter l’état du ticket
- Ajouter des commentaires
- Répondre aux questions du support


## 5. Bonnes pratiques pour l’utilisateur

- Être précis et factuel
- Choisir la bonne catégorie
- Indiquer les impacts (bloquant ou non)
- Mettre à jour le ticket si la situation évolue

## 6. Résultat attendu

- Ticket clair et exploitable
- Prise en charge rapide par le support
- Communication fluide entre utilisateur et technicien
